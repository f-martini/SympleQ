Contributing
============

Thanks for your interest in contributing to SympleQ! This document provides
guidelines for contributing to the project, including how to report issues, submit code changes, and adhere to the project's coding standards.


Reporting Issues
----------------

If you encounter any issues while using SympleQ, please report them by creating a new issue in the project's GitHub repository. When reporting an issue, be sure to include the following information:

1. A clear and descriptive title
2. A detailed description of the issue, including steps to reproduce it
3. Any relevant error messages or logs
4. Your environment details (e.g., operating system, Python version)


Submitting Code Changes
-----------------------

We welcome contributions to SympleQ! To submit code changes, please follow these steps:

1. Fork the repository and create a new branch for your changes
2. Make your changes and add tests to cover them
3. Ensure that all tests pass
4. Submit a pull request with a clear description of your changes

If at any point you need help or have questions, feel free to reach out to the project maintainers or the community via 
the project's communication channels.

If your work is related to a specific issue, please reference it in your pull request description.  

.. note::
   The suggested PEP 8-related extensions for Visual Studio Code (see
   `Visual Studio Code Extensions <development/extensions.html>`_), automatically
   detect PEP 8 violations highlighting them in the editor as warnings
   and providing suggestions to fix them.


Coding Style
------------

The coding style adopted for this project is **PEP 8**. PEP 8 is the style guide
for Python code, which helps to maintain readability and consistency across the
codebase. While not mandatory, it is recommended to follow the this coding
standard.

In `this page <https://www.python.org/dev/peps/pep-0008/>`_ it is possible to browse the full PEP 8 style guide.
